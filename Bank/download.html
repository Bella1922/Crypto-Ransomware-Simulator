Overview of the download and encryption phases

The download process in this demo starts when a victim visits a

fake bank page and clicks a link to “download” their bank statement.

That file is actually a malicious payload disguised as a legitimate

document; when opened it executes a downloader that fetches and launches

the main ransomware program and then embeds itself on the system. This

mimics the common social-engineering vector where users are tricked into

running what they think is a harmless file.


Next, in the encryption process, the ransomware would scan local and network

drives for targeted file types, attempt to disable recovery tools and remove

backups, then use a fast symmetric algorithm to encrypt files so they're

inaccessible without the decryption key. Everything shown here is conceptual

and safe, no real encryption occurs and your files are not harmed.
